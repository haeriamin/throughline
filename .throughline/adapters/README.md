# Adapters: one source, many tools

Throughline is "one brain, N adapters." The brain lives under `.throughline/` — constitution,
runbooks, personas, commands, instructions, skills, hook scripts, agent definitions, and global
rules. Each AI coding tool only needs a thin layer of *wiring* that points its own persona,
command, hook, and rules files at that shared brain.

That wiring is **generated**, not committed. You edit the source under `.throughline/adapters/source/`;
`tools/convert` renders the per-tool files locally. After clone, run `tools/install` once before
using any runtime.

## Layout

```
.throughline/adapters/
    source/
    personas/*.persona       # metadata (tools, sandbox, description); @@BODY@@ is fallback only
    agents/*.agent.md        # canonical persona + Copilot command agent bodies
    commands/*.command        # one per slash command
    instructions/*.instructions.md
    skills/*/SKILL.md
    hooks/*.sh                # shared hook scripts (copied to .github + .claude)
    tool-docs/                # runtime README + VERIFICATION (codex, cursor, etc.)
    globals/                  # AGENTS.md, CLAUDE.md, copilot-instructions, settings
    global-rules.md           # non-negotiables + confidence gates
    hook-spec.tsv             # hook wiring table
  profiles/*.profile          # one per tool: emit targets and formats
  generated/                  # staged templates (e.g. Cursor hooks template) — also generated
  README.md                   # this file
```

Generated adapter files land in each tool's directory (`.claude/`, `.codex/`, `.cursor/`, …) plus
`.github/agents/`, `.github/instructions/`, `.github/skills/`, root `AGENTS.md` / `CLAUDE.md`, and
manifests under `.throughline/integrations/`. They carry a "generated by tools/convert" marker — **do
not edit them by hand**; edit the source and regenerate.

## Running the generator

```bash
# After clone — pick your tool(s) and wire hooks:
bash tools/install.sh              # interactive
bash tools/install.sh --tool cursor
bash tools/install.sh --all

# Or call convert directly:
bash tools/convert.sh --tool all   # shared content + every tool
bash tools/convert.sh --tool shared  # GitHub agents/instructions/skills/hooks + root globals only
```

On Windows without Git Bash, use `tools/install.ps1` / `tools/convert.ps1` — output must match
`convert.sh` byte-for-byte (CI enforces parity).

## CI

CI does **not** expect generated files in git. It runs `convert.ps1`, checksums the output, runs
`convert.sh`, and fails if the two generators disagree.

## Adding a new tool

1. Add `profiles/<tool>.profile` (copy the closest Tier A or B profile).
2. If the tool needs a new output format, add a branch to `Emit-*` in **both** `convert.ps1` and
   `convert.sh`.
3. Run `tools/convert --tool <tool>`, verify locally, document in `docs/runtimes/`.
